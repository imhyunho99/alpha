## 🛠 기술 스택

### Backend
- `Python/FastAPI` - RESTful API 서버
- `QuestDB` - 시계열 데이터베이스
- `yfinance` - 금융 데이터 수집

### AI/ML
- `scikit-learn/XGBoost/LightGBM` - 앙상블 머신러닝 모델
- **특성 엔지니어링**: 13개 기술적 지표 (SMA, RSI, MACD, Bollinger Bands 등)
- **앙상블 기법**: Soft Voting (3개 모델 조합)

### Frontend
- `PySide6` - Qt 기반 데스크톱 GUI

### DevOps
- `Git/GitHub` - 버전 관리
- 환경변수 기반 보안 관리

## 🔧 기술적 도전과 해결

### 1. 모델 성능 개선 (55% → 71.79%)

**문제점**
- 초기 모델(RandomForest)의 정확도가 55-58%로 낮음
- 4개의 단순한 기술적 지표만 사용

**해결 방법**
- **Phase 1**: 특성 확장 (4개 → 13개)
  - SMA_20, SMA_50, EMA_12, RSI_14, MACD, Bollinger Bands 등 추가
  - 하이퍼파라미터 튜닝 (n_estimators: 100→200, max_depth: 15)
  
- **Phase 2**: 앙상블 모델 도입
  - RandomForest + XGBoost + LightGBM 조합
  - Soft Voting으로 3개 모델의 확률 평균 사용
  - 단일 모델 대비 안정성 및 정확도 향상

**결과**
- 정확도 **27.1% 개선** (55% → 71.79%)
- 예상 수익률 S&P 500 대비 **+9.79%p**

### 2. 대용량 데이터 처리

**문제점**
- 206개 자산 × 3년치 일별 데이터 (약 150만 행)
- 실시간 업데이트 시 성능 저하

**해결 방법**
- **QuestDB 도입**: 시계열 데이터 전용 DB로 쿼리 속도 10배 향상
- **배치 처리**: 자산별 병렬 처리로 학습 시간 단축
- **모델 캐싱**: 학습된 모델을 `.joblib` 파일로 저장하여 재사용

**결과**
- 전체 자산 업데이트 시간: **30분 → 5분**
- 모델 학습 시간: **2시간 → 20분**

### 3. 보안 관리

**문제점**
- DB 비밀번호 하드코딩
- 민감한 데이터 파일이 Git에 포함될 위험

**해결 방법**
- 환경변수(`.env`) 기반 설정 관리
- `.gitignore` 강화 (환경변수, 모델 파일, 데이터 파일 제외)
- `.env.example` 제공으로 설정 가이드 제공

**결과**
- 민감 정보 완전 분리
- GitHub 안전하게 공개 가능

## 🎓 배운 점 및 느낀 점

### 기술적 성장
1. **앙상블 학습의 중요성**
   - 단일 모델보다 여러 모델을 조합하면 안정성과 정확도가 크게 향상됨을 경험
   - Soft Voting 기법으로 각 모델의 장점을 효과적으로 결합

2. **특성 엔지니어링의 중요성**
   - 단순히 모델을 복잡하게 만드는 것보다 의미있는 특성을 추가하는 것이 더 효과적
   - 금융 도메인 지식(기술적 지표)을 ML에 적용하는 방법 학습

3. **시계열 데이터 처리**
   - QuestDB를 통해 시계열 데이터 전용 DB의 성능 이점 체감
   - 대용량 데이터 처리 시 적절한 DB 선택의 중요성 인식

### 프로젝트 관리
1. **단계적 개선의 중요성**
   - Phase 1, 2, 3로 나누어 점진적으로 개선하며 각 단계의 효과 측정
   - 문서화를 통해 개선 과정과 결과를 체계적으로 기록

2. **보안의 중요성**
   - 개발 초기부터 보안을 고려하지 않으면 나중에 수정하기 어려움
   - 환경변수 관리, .gitignore 설정 등 기본적인 보안 수칙의 중요성
